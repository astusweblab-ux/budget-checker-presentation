Budget Checker 2.0

Платформа для проверки смет и списков материалов: пользователь загружает файл или вводит строки, система находит реальные предложения магазинов и показывает explainability (почему матч случился) + помечает спорные строки.

Коротко о продукте

Ценность: быстро понять «сколько реально стоит» список материалов/работ по рынку, и в каких местах смета неочевидна (needs_review).

  • Публичный сценарий: загрузка сметы → polling статуса → результаты с explainability.
  • Merchant сценарий: загрузка каталога офферов (CSV/XLSX / ingestion adapters) → офферы участвуют в compare.
  • Ops/Admin сценарий: верификация магазинов и модерация статусов с audit trail.

Проблема

  • Сметы на ремонт/строительство сложно проверять: позиции размыты, а итоговая стоимость непрозрачна.
  • Цены на материалы сильно отличаются по магазинам и регионам — ручное сравнение занимает часы.
  • Подрядчики могут «прятать» маржу в материалах и работах, а клиент не видит, где переплата.
  • Магазины и подрядчики получают много холодного трафика и мало доверия, потому что нет прозрачного сравнения.

Решение

  • Пользователь загружает смету (сейчас: CSV/XLSX) или вставляет список строк.
  • Система находит реальные предложения магазинов и показывает альтернативы, цены и диапазоны.
  • Каждая строка получает confidence_score и match_reason, спорные — needs_review.
  • Партнёры загружают каталог через кабинет (CSV/XLSX) или через интеграции (feed/API) и проходят верификацию.
  • Результат фиксируется snapshot-ом: пользователь видит расчёт «как было» на момент проверки.

Как это работает (упрощённо)

  • Данные: нормализованный каталог (material_items/service_items) + реальные офферы партнёров.
  • Ingestion: импорт в офферы + история цен (retail_offer_price_snapshots) + валюты (currency_rates).
  • Compare pipeline: POST создаёт job → обработка → GET polling → UI показывает результат.
  • Матчинг: baseline-алгоритм с explainability (токены/числа/категории) и порогом уверенности.
  • Доступ: RBAC, проверки границ организации, audit trail для модерации и критичных действий.

Ценность для пользователя

  • Быстро понять «сколько это должно стоить» по рынку и где есть альтернативы.
  • Видеть спорные строки и править их вручную, вместо слепого доверия «чёрному ящику».
  • Сравнить закупку по магазинам/филиалам с учётом локации и наличия.
  • Снизить риск переплаты и получить понятный отчёт для принятия решения.

Ценность для партнёров

  • Высокоинтентный трафик: пользователь уже имеет список материалов и готов к покупке.
  • Прозрачная витрина: верификация магазина повышает доверие и качество сравнения.
  • Автосинхронизация каталога через интеграции снижает операционные затраты.
  • В перспективе: отчёты спроса, качество данных, лиды и аналитика по регионам.

Монетизация (план)

  • CPA / revenue share: клики и переходы к партнёрам, заявки, покупки.
  • Подписка для партнёров: интеграции, автосинк, отчёты качества каталога, аналитика спроса.
  • Платные пакеты для подрядчиков: лиды и benchmark по рынку (на базе прайс-листов).
  • Enterprise: отчёты/интеграции под крупные сети и девелоперов.

Монетизация — фаза 8 по docs/ROADMAP.md. Сейчас фокус: данные, качество и интеграции 2 сетей.

Почему мы отличаемся

  • В центре продукта — реальные офферы рынка, а не ограниченный «core-список» позиций.
  • Explainability по каждой строке: видно, почему совпало и насколько уверенно.
  • Async-ready архитектура: pipeline готов к росту объёма строк и офферов.
  • Snapshot результата: воспроизводимость, аудит и отсутствие «переезда» цен задним числом.
  • Security baseline встроен в архитектуру: RBAC, audit trail, разделение публичного и admin контуров.

Ближайшие вехи (90 дней)

  • Epicentr: довести structured feed ingestion до реального партнёрского режима + регулярная синхронизация.
  • Nova Liniya: второй адаптер (partner API) и тот же pipeline офферов.
  • Метрики качества источников: ошибки импорта, свежесть данных, health/status.
  • Golden test set для матчинга и улучшение ранжирования без регрессий.
  • Первые пилоты с 1–2 партнёрами и реальными каталогами.

Сквозной demo-flow (локально)

  • Запуск: start_project.bat.
  • Открыть: http://localhost:3000/dev/routes (живые URL + demo-логины).
  • Merchant: импорт каталога → admin: verify shop → публичный upload сметы → compare результат.

Фаза 2 — Auth, роли и организации ГОТОВО

  • RBAC: system roles + org roles, guard-ы от privilege escalation и IDOR.
  • Организации: members, invites по email, audit trail для admin действий.
  • Merchant: Shop/Branch + verification_status, ContractorProfile foundation.
Фаза 3 — Материальный каталог V1 ГОТОВО

  • Import pipeline: ImportJob/ImportJobRow, idempotency по external_id в рамках branch.
  • Валюта: currency_rates, requires_currency_rate, price_uah.
  • История цен: retail_offer_price_snapshots (пишем только при реальном изменении).
  • UI: partner import page + admin shop verification page.
Фаза 4 — Матчинг и compare материалов ФУНКЦИИ ГОТОВЫ / НЕТ ДАННЫХ

  • Estimate upload/polling/correction + UI: / и /estimates/[jobId].
  • Критерий готовности: нужен минимум 2 источника реальных предложений (закрывается в Фазе 5).
